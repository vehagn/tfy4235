Spin Glass

The Ising spin glass is described by the Hamiltonian

$$ H = -\sum_{\left< i,j \right>} a_{i,j} S_{i} S_{j} $$

where $a_{i,j} = \pm 1 $ with some probability distribution, usually flat, and describe the kind of interaction.

$a_{i,j} = 1$ means it's a ferromagnetic interaction, while $a_{i,j} = -1$, means it's an anti-ferromagnetic interaction.

$\sum_{\left< i,j \right>} $ means the sum over the nearest neighbors nodes.

We'll look at a regular 2D lattice in our case.

Representation

We need some way to represent the spins. Since a spin can only have one of two possible states, we can choose $0$Â to represent spin down and $1$ to represent spin up. This way we can pack several spins in a single integer, and save memory for large systems, and do some calculation in parallel. Since the spins is represented as bits, we can represent the interactions using binary operations:

$$S_i S_j = 1 - 2 \times XOR(S_i,S_j)$$

To see this, consider the two cases:

  1. $S_i$Â and $S_j$ are different, then $XOR(S_i,S_j) = 1$, and $S_i S_j = -1$ .
  2. $S_i$Â and $S_j$ are identical, then $XOR(S_i,S_j) = 0$, and $S_i S_j = 1$ .

Genetic algorithm

To use a genetic algorithm we need

  1. a suitable representation of the system
  2. a fitness function to select for

The first condition is already satisfied if we use the bit representation above. The Hamiltonian satisfies the second condition. The procedure is then as follows

  1. Initialize the system by generating N individuals which represent possible states.
  2. Generate new individuals by using some scheme to select the most fit individuals from the previous generation with a given probability.
  3. Do the crossover-step on the new generation, mixing the genes of random individuals with a given probability.
  4. Mutate individual bits with given probability.

The choice of probabilities and the size of the population decides how fast it converges, if it converges at all.

Simulated Annealing

Reference: Kirkpatrick, et al., Science, New Series, Vol. 220, No. 4598. (May 13, 1983)

We want to minimize the function $E(\vec x)$. We treat the function as the energy of a thermal system with temperature T, with probability

$$ p(\vec x) \propto e^{-E(\vec x)/T} $$

where $\vec x$Â can be any dimension. Then we use a Monte Carlo method to generate a sequence of states. If the temperature then more states will be likely to be visited a lot. If we slowly decrease the temperature, the state will eventually converge to the global minimum.

Scheme:

Start by having an ensemble of states, n different initial configurations, and follow the ensemble average of the energy, $\left< E \right>$.

  1. Set an initial temperature, $T_0$.
  2. Make $M$Â Monte Carlo steps (one sweep), where $M$Â is the number of degrees of freedom.
  3. Let $\left< E \right>_j$Â be the ensemble average of the energy after $j$Â Monte Carlo sweeps.
  4. if $\left< E \right> _ j \gt \left< E \right> _ {j-1}$, go to II.
  5. else continue.
  6. Adjust temperature: $T_ {j+1} \to c T_j, ~c \lt 1$
  7. Go to II if maximum number of Monte Carlo sweeps has not been reached.

Pseudocode:

initialize() // intialize spins and couplings, and calculate energy

for (j = 0; j < jmax; j++)
    while (E_current > E_previous) // stops if the new average energy is better
        monteCarloSweep()          // M Monte Carlo steps for each sample, and calculated the new average energy
    endwhile
    T = T * c
    E_previous = E_current
endfor

Some of the copies in the ensemble tend to get stuck in local minima as we lower the temperature.

Simulated Tempering

Reference: Article by Parisi and Marinari.

The method of Simulated Tempering differs from Simulated Annealing in that the temperature of each copy in the ensemble can be different. We therefore also do a Monte Carlo update of the temperature:

Define $\beta_i = \frac{1}{T_i}, ~T_i \in {T_1,...,T_m}$

In the constant temperature case, we have the Boltzmann factor

$$ w(E_\alpha) \propto e^{- \beta E_\alpha}$$

which we can generalize to

$$ w(E_\alpha,\beta_\alpha) \propto e^{- \beta_\alpha E_\alpha + g(\beta_\alpha)}$$

where $g$Â is a function that controls the temperature distribution. Different choices of $g$Â gives different performance. The most optimal form is to set $g(\beta_\alpha) = \beta_\alpha f$, where $f$Â is the free energy, which we do not know.

Scheme:

For a given sample in the ensemble, and given a choice of $g$:

  1. Trial move in temperature space, $\beta_i \to \beta_{i+1}Â or ~ \beta_{i-1}$.
  2. Calculate $\Delta g _ {i,i\pm1} = g(\beta_{i\pm1}) - g(\beta_{i})$.
    • If $\Delta g _ {i,i\pm1} > 0$, accept move.
    • Otherwise, generate a random number $r$.
      • if $e^{-\Delta g _ {i,i\pm1}} \le r$, accept move
      • else, reject.
  3. Perform standard Monte Carlo move in configuration space.
